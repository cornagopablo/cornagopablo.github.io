Criptografía simétrica y asimétrica

Criptografía simétrica

Este método utiliza una misma clave para cifrar y descifrar, por ello emisor y recptor deben conocer previamente la clave o enviársela, a través de un canal seguro.

Este sistema está en desuso básicamente por dos motivos:

  • El problema es conseguir un intercambio seguro de claves.

  • El número de claves necesarias para comunicarse con varias personas se corresponde con la fórmula: N*(N-1)/2

En términos de seguridad, en este tipo de cifrado, toda la seguridad está reflejada en la clave. Por lo que compartirla representa una gran vulnerabilidad. Sin embargo, cuando se emplea este método se deben cumplir dos parámetros esenciales para que se considere seguro, los cuales son:

  1. Después de encriptar una información, no se puede obtener la clave usada para el cifrado y el descifrado.
  2. El costo de descifrar una información debe ser más alto que la misma información contenida en el mensaje cifrado.
Para descifrar el mensaje se emplean los Ataques de Fuerza Bruta, que consiste en aplicar un método de prueba y error para dar con la convinación correcta.

A pesar de todo lo dicho, este tipo de criptografía representa varias ventajas. Por ejemplo, la rapidez, ya que requiere de menor poder computacional debido a la longitud de sus claves, que son de 64 bits. Además, posee una infraestructura sencilla y sólo requiere de una clave, por lo que es muy fácil de emplear para el cifrado de archivos que contengan datos personales.

Por otro lado, el cifrado simétrico garantiza la privacidad y la integridad en las comunicaciones vía telefónica o por Internet, como el caso del correo electrónico, pero no la irrefutabilidad entre ambos.



Criptografía asimétrica

Ante los problemas presentados con el uso de la criptografía simétrica, en 1976 dos americanos idean un sistema de comunicación en el que cada una de las partes tiene una pareja de claves.

Este sistema consiste en la utilización de una fórmula matemática muy compleja para crear un par de claves. Esta primera clave es la clave privada. La clave privada es de uso exclusivo para el creador del par de claves y sirve para cifrar y descifrar mensajes de forma completamente segura.

La segunda clave es llamada clave pública. Esta es una clave que el creador puede entregar a terceras personas. La clave pública se crea a partir de la clave privada, pero el proceso inverso es imposible. De esta forma, el creador de claves puede compartir esta clave pública con terceras personas y gracias a ella, estas personas pueden enviarle información cifrada que solo será accesible usando la clave privada del creador.





Las principales ventajas de este cifrado son:

  1. Este sistema ofrece una alta tasa de seguridad, pues el esquema de cifrado es altamente complejo. El criptoanálisis de estos sistemas es complicado y los ataques de fuerza bruta para romperlo son insuficientes y nada prácticos.

  2. Permite asegurar canales de comunicación abiertos y públicos. Esto permite que emisor y receptor puedan compartir información de forma segura.

  3. Ofrece un alto nivel de confidencialidad, integridad y garantiza el no-repudio de la información


Las desventajas, podemos citar:

  1. En comparación con los sistemas de cifrado simétricos, es computacionalmente más costoso.

  2. La complejidad de los algoritmos resulta en una dificultad en el análisis y seguridad de los mismos. Esto hace que detectar fallos o bugs sea más complejo y difícil en estos sistemas.


Por ello, una buena fórmula de ecriptación es la mezcla entre las dos, consiste en:

  1. Criptografía simétrica para enviar los mensajes.

  2. Criptografía asimétrica para enviar las claves.
